# ComfyUI Telegram 机器人 - AI图像处理系统

一个基于ComfyUI的Telegram机器人，支持AI图像处理和批量任务管理。

## 🔧 功能特性

- **AI图像处理**: 通过Telegram Bot接收图片并使用ComfyUI进行AI处理
- **智能队列管理**: 支持多用户并发任务，自动排队处理
- **实时状态监控**: 图形化界面显示处理进度和统计数据
- **代理自动检测**: 自动识别并配置系统代理设置
- **任务历史记录**: 支持重新生成历史任务
- **统计数据分析**: 实时展示处理量、耗时等关键指标

## 🚀 快速开始

### 环境要求

- Python 3.8+
- ComfyUI 服务运行中
- Telegram Bot Token

### 安装依赖

```bash
pip install -r requirements.txt
```

### 配置说明

#### 1. 创建Telegram Bot

1. 在Telegram中搜索 [@BotFather](https://t.me/BotFather)
2. 发送 `/newbot` 命令
3. 按提示输入bot名称和用户名
4. 获取到类似这样的Token：
   ```
   8678.............6:AAHfZ-..........................
   ```

#### 2. 配置Bot Token

在 `bot.py` 文件中找到以下配置项并修改：

```python
# ================= 配置区域 =================
BOT_TOKEN = '你的Telegram Bot Token'  # 替换为你的实际Token
COMFYUI_SERVER = "127.0.0.1:8000"    # ComfyUI服务地址
COMFYUI_DIR = r"D:\C"                 # ComfyUI安装目录
```

#### 3. ComfyUI工作流配置

确保你有一个名为 `workflow_api.json` 的工作流文件，该文件定义了AI处理的具体流程。

### 启动服务

```bash
python bot.py
```

启动后会看到图形化控制界面，显示实时处理状态。

## 📱 使用方法

### 用户操作指南

1. **开始使用**: 在Telegram中向你的Bot发送 `/start` 命令
2. **查看状态**: 发送 `/status` 或点击"📊 刷新状态"按钮
3. **处理图片**: 直接发送图片给Bot即可自动加入处理队列
4. **重新生成**: 在处理结果中点击"🔄 重新生成"按钮

### 系统限制

- 每个用户最多同时处理 **5个任务**
- 队列会自动管理任务优先级
- 支持断线重连和错误恢复

## 🛠️ 技术架构

### 核心组件

```
bot.py              # 主程序文件
├── Telegram Bot    # 用户交互接口
├── ComfyUI Worker  # AI处理引擎
├── Queue Manager   # 任务队列管理
└── GUI Interface   # 图形化监控界面
```

### 数据流向

```
用户发送图片 → Telegram Bot → 任务队列 → ComfyUI处理 → 结果返回用户
     ↓              ↓            ↓           ↓            ↓
   图片接收      队列管理      AI计算      结果生成      用户反馈
```

### 关键技术点

- **多线程处理**: 使用QThread实现非阻塞的GUI和处理逻辑
- **WebSocket通信**: 与ComfyUI建立实时连接
- **智能代理**: 自动检测系统代理设置
- **状态持久化**: 统计数据自动保存到JSON文件

## 📊 监控面板说明

图形界面包含以下监控信息：

- **等待队列**: 当前排队中的任务数量
- **今日完成**: 今天已处理的任务总数
- **昨日统计**: 昨天的处理量对比
- **近30天总量**: 月度处理趋势
- **平均生成耗时**: 单个任务的平均处理时间

## 🔧 高级配置

### 代理设置

程序会自动检测常见代理端口：
- Clash: 7890
- V2Ray: 10809, 10808
- Shadowsocks: 1080

如需手动配置，在代码中修改 `auto_setup_proxy()` 函数。

### 性能调优

```python
MAX_USER_TASKS = 5  # 每用户最大并发任务数
# 可根据服务器性能调整此值
```

### 日志级别

程序会在控制台输出详细的操作日志，包括：
- 系统启动状态
- 任务处理进度
- 错误和异常信息
- 网络连接状态
